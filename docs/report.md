# 五子棋课程设计报告

## 1. 选题描述

本项目选择课程设计第二选题“五子棋博弈”。题目要求先完成一个可正常运行的五子棋程序，实现棋盘表示、落子合法性判断、胜负判定、基础交互流程与简单智能体。在此基础上，阶段二再继续实现局面评估、博弈树搜索与剪枝优化。

阶段一的核心目标不是追求高棋力，而是先把环境、规则和对弈流程搭建完整，并保证程序能够稳定运行，为后续搜索算法提供可靠基础。

## 2. 项目实现方案设计

本项目采用 Python 实现，界面选择标准库 `curses`，没有使用 `textual` 等第三方 TUI 框架。这样做的主要原因是依赖少、状态刷新路径更简单，终端中的重绘行为更容易控制，稳定性更高。

整体方案分为四层：

- 规则层：负责棋盘状态、合法性判断、连五检测、平局检测
- 流程层：负责回合推进、落子应用、终局处理
- 策略层：负责人类玩家与 AI 的下一步决策
- 界面层：负责主菜单、棋盘渲染、键盘和鼠标输入

阶段一界面包含两个页面：

- 主菜单：选择 `P1` 类型、棋盘大小、启动游戏、导出数据、退出
- 对局界面：显示棋盘、当前回合、返回菜单、退出

为了方便阶段二扩展，游戏流程与 AI 完全解耦。当前 baseline AI 只通过统一接口返回落子位置，后续可以直接替换为 Minimax、Alpha-Beta 等搜索式 AI，而不需要重写规则和界面。

对局数据采用结构化保存方案：

- 单局保存为 JSON
- 批量导出为 JSONL 和 CSV

这样既方便复现某一局棋，也方便后续统计实验结果。

## 3. 项目架构与模块说明

项目主要代码位于 `gomoku/` 目录中。

### 3.1 核心模块

- `app.py`
  - 程序入口，负责启动 `curses` 主循环。

- `models.py`
  - 定义核心数据结构与枚举，包括棋子类型、玩家类型、对局结果、配置、落子记录、游戏状态。
  - `GameState.current_round` 实现了“黑白双方各走一步算一回合”的回合计算。

- `rules.py`
  - 实现棋规相关逻辑，包括边界判断、落子是否合法、坐标转换、连五检测、邻域候选点生成。
  - 这里的算法主要是沿四个方向统计连续棋子数量，用于判断是否形成五连。

- `engine.py`
  - 实现对局流程控制。
  - `MatchEngine.apply_move()` 负责应用一步落子，更新回合、记录历史、判断胜负与平局。

- `players.py`
  - 定义玩家控制器接口和 AI 接口。
  - `BaselineAI` 实现阶段一规则式 AI，其决策顺序为：
    - 先找立即获胜点
    - 再找对手的一步成五并封堵
    - 然后在已有棋子邻域中枚举候选点并评分
  - 候选点评分使用简单启发式，综合考虑自身棋形、防守价值、邻接度和中心偏置。
  - 同时预留了 `PredictionProvider`、`Evaluator`、`CandidateGenerator`、`SearchPolicy`、`SearchAI` 等接口，对应阶段二的搜索扩展。

- `ui.py`
  - 实现完整终端界面。
  - 包括主菜单渲染、棋盘渲染、键盘输入、鼠标输入、菜单按钮、状态栏、面状数字字体显示。
  - 棋盘使用高亮格表示当前选中位置，支持方向键与鼠标两种操作方式。

- `persistence.py`
  - 负责保存已结束对局。
  - 每局保存棋盘大小、玩家类型、时间戳、结果、终局原因、完整落子序列和最终棋盘快照。

- `exporter.py`
  - 负责扫描历史 JSON 对局文件并导出为 `games.jsonl` 与 `moves.csv`。

### 3.2 测试模块

- `tests/test_engine.py`
  - 对规则引擎、回合数、胜负判定、baseline AI、数据保存与导出进行单元测试。

## 4. 最终实现效果

目前项目已完成阶段一要求，并实现了以下效果：

- 可以正常运行完整的五子棋程序
- 支持 `9~19` 的奇数尺寸棋盘，默认 `15x15`
- 黑棋固定先手
- 支持边界检测、重复落子检测、五连胜负判定和平局判定
- 支持人机对战与 AI 对 AI
- 支持键盘与鼠标两种交互方式
- 主菜单与对局界面均已实现
- 主菜单标题与回合数采用 Unicode 块字符绘制
- baseline AI 可稳定输出合法落子，不使用纯随机策略
- 已结束对局可保存为 JSON，并可批量导出为 JSONL / CSV

从工程结构上看，当前版本已经具备阶段二继续扩展的基础。后续只需要在现有接口上补充局面评估函数、候选点筛选、Minimax 与 Alpha-Beta 剪枝等搜索逻辑，即可继续完成高级博弈算法部分。
